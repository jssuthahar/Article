
# Azure Logic Apps: Editing Workflows

## üìå What are Azure Logic Apps?

Azure **Logic Apps** let you **automate workflows** by connecting apps, data, and services without writing much code.
Think of it like a **visual workflow builder** for integrating services (e.g., Outlook, Salesforce, SQL, APIs).

Two main types of workflows:

* **Standard workflows** ‚Üí Automating everyday business tasks.
* **Enterprise workflows (B2B)** ‚Üí Complex cross-organization workflows, such as **EDI (Electronic Data Interchange)**, using **Enterprise Integration Pack (EIP)**.

---

## üìå Editing Workflows in Logic Apps

When you need to edit workflows, you have different tools:

1. **Logic Apps Designer** ‚Üí Drag-and-drop visual editor.
2. **Logic App Code View** ‚Üí Edit JSON definition of workflows.
3. **Enterprise Integration Pack (EIP)** ‚Üí For **B2B / enterprise integration workflows**.
4. **API Connections** ‚Üí Define and manage connections to external services (not for editing workflows).

---

## üìå Question 1 ‚Äì Editing Enterprise Workflows

**Scenario:**
You are a developer at your company.
You need to **edit the workflows for an existing Logic App**.

**Options:**
A. The Enterprise Integration Pack (EIP)
B. The Logic App Code View
C. The API Connections
D. The Logic Apps Designer

**‚úÖ Correct Answer: A ‚Äì Enterprise Integration Pack (EIP)**

* **Why correct?**

  * EIP is required for **enterprise-level (B2B)** workflows.
  * Supports **EDI, AS2, X12 protocols**.
  * Designed for **large-scale integration across organizations**.

**‚ùå Why others are wrong:**

* **B. Code View** ‚Üí For editing JSON, but not specific to enterprise workflows.
* **C. API Connections** ‚Üí Only manages connections, doesn‚Äôt edit workflows.
* **D. Designer** ‚Üí Good for standard workflows, not for enterprise/B2B integration.

---

## üìå Question 2 ‚Äì Updating Workflow Definitions

**Scenario:**
You are a developer at your company.
You need to **update the definitions** for an existing Logic App.

**Options:**
A. The Enterprise Integration Pack (EIP)
B. The Logic App Code View
C. The API Connections
D. The Logic Apps Designer

**‚úÖ Correct Answer: B ‚Äì Logic App Code View**

* **Why correct?**

  * **Code View** allows editing the Logic App **definition directly in JSON**.
  * Used when you need to tweak or update workflow logic beyond what Designer supports.

**‚ùå Why others are wrong:**

* **A. EIP** ‚Üí Only for enterprise workflows, not standard definition editing.
* **C. API Connections** ‚Üí Just manages external connections.
* **D. Designer** ‚Üí Visual editing, but JSON definition updates require Code View.

## üìå Key Learning for Beginners

* **Designer** = Visual editing (best for beginners).
* **Code View** = JSON editing (for developers).
* **EIP** = Enterprise integration (B2B scenarios).
* **API Connections** = Define/manage service connectors.

---

‚úÖ **Industry Takeaway:**

* Use **Designer** when building simple workflows.
* Use **Code View** for JSON definition updates.
* Use **EIP** when building **enterprise-level B2B workflows**.
